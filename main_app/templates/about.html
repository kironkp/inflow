About InFlow

A small flowchart maker for people who think in branches and arrows but hate aligning boxes.

Why I built this

Every other flowchart tool starts with an infinite canvas, a snap-grid, and a fight to keep boxes from drifting. That's the wrong starting point for thinking. A flow has a shape — top to bottom, parent to child, with the occasional branch — and that shape is a tree.

InFlow treats your flowchart as a tree of nodes. You add a child, the layout does the math. You drag a node onto another, the connection comes with it. The result reads like a real flowchart because it is one — just one you didn't have to position by hand.

How it works

  • Flowcharts are containers. Each one is a separate diagram with its own tree of nodes.
  • Nodes are shapes — Process, Decision, Start/End, Input/Output, Data, or Note. Each can have a parent (another node) and any number of children.
  • Branch labels sit on the connector from a parent to a child. Use them for "Yes" / "No" from a decision, or any short label that says what the branch means.
  • Drag a node onto any other node to reparent it. Drop it on the canvas to make it a new root.
  • Tags are a shared catalog you can sprinkle on nodes — handy for "manual step", "approval needed", "risk".

Tech & credits

  • Backend: Django + PostgreSQL (SQLite fallback in dev)
  • Frontend: plain Django templates + hand-written CSS Grid & Flexbox
  • Layout: family-tree CSS adapted from org-chart conventions
  • Images: hand-rolled SVG illustrations
  • Type: Inter + Fraunces via Google Fonts
